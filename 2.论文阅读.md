# 论文阅读

**纸上得来终觉浅，绝知此事要躬行。本文所有内容均为参考，具体实施需因时而异**

> 其中本文大体是对《How to read a paper》一文的翻译，结合了少量其他前辈的经验

> 知乎《如何阅读论文》一文除了介绍了"三步走"的论文阅读方法，引用了相当数量的前辈经验，非常详细地介绍了不同情况下的论文阅读方法，还有关于如何构思Idea的介绍，强烈推荐

读论文主要有两个目的：

1.  了解某一研究领域的现状与发展趋势

2.  启发和构思自己的研究想法（idea）

因此，研究者会花费相当多时间阅读论文。但从头到尾阅读文章是极其耗时耗力的，也是不可取的。

借鉴前人的思想，于此介绍"three-pass"的论文阅读方法。

> 每一次阅读文章都要达成特定的目标，每一次的阅读都是在前一次阅读的基础上进行的

## The first pass

> 核心：了解论文的大概思路（是什么、为什么、贡献在哪里）

**做法**：

1.  阅读标题、摘要、引言、每个小节的标题（忽略其中的细节）、结论

2.  浏览参考文献，标记读过的工作

完成上述工作之后，要能回答以下5个问题**(five Cs)**：

1.  Category：这篇文章是什么类型的？

2.  Context：这篇文章相关的工作有哪些？分析文中问题需要哪些理论基础？

3.  Correctness：文章的假设是有效的吗？

4.  Contributions：论文的主要贡献是什么？

5.  Clarity：作者是否清晰、有条理地传达了他们的研究工作？

完成第一步阅读之后，应该能对文章解决什么样的问题有一个了解。这时可以判断文章是否对自己的研究工作存在价值，是否需要进一步阅读。

这一层的目标是**建立总体印象，并判断是否值得投入时间进行更深入的阅读**

## The second pass

> 核心：了解主要技术内容和论证思路，不追求所有细节（证明、代码、数学推导）

做法：

1.  仔细查看论文中的图表，检查是否存在一些常见错误（坐标轴是否正确标注？结果是否可靠、**有统计意义？**）

2.  标记参考文献中尚未了解的工作（一个很好的可以进一步了解这篇论文背景的方法）

完成上述工作后，应该能够把握文章的主要内容（提出了什么，为什么这么做，用了什么，结果怎样），**具备向别人清楚概括论文核心思想的能力**，并且能够用论文里的证据支持

如果发现完成上述工作后，依然不能理解论文的主要内容，可能有几个原因：

1.  对论文主题不了解，对术语和缩写陌生

2.  作者使用了不熟悉的证明方法或实验手段

3.  论文写作质量差，存在没有论据支撑的断言以及频繁的前向引用

4.  ~~你累了~~

这个时候可以选择：

1.  把论文扔在一遍，~~祈祷在你的科研生涯中不需要理解这篇文章~~

2.  之后再看，比如了解背景资料后再看

3.  继续第三步

## The third pass

核心：完全了解一篇工作

论文原话：**[The key to the third pass is to attempt to virtually
re-implement the paper: that is, making the same assumptions as the
authors, re-create the work.]{.underline}**

通过对比re-creation和实际的工作，发现工作中的创新点以及潜藏的不足和假设

做法：仔细地阅读每一处细节，发现并质疑作者每个论点中的假设。思考如何去表述自己的观点，深入理解论文的论证方法、证明思路和写作技巧。在这一遍阅读当中，还应该记录想到的未来研究方向和改进的思路。

完成第三遍的阅读工作后，应该能复原整篇论文的结构，清楚指出论文的优点和缺点，发现作者隐藏的假设、没有引用的重要相关工作，批判地评价其方法、论证与写作技巧

一般来说，读完论文之后写笔记是很有必要的。笔者认为论文笔记除了有对文章工作的自我表述之外（非必需），还应该记录从论文中学习到的东西（写作技巧、idea构思、系统设计、实验设计等方面），其他要素因人而异。

笔记形式、类型多样，只要遵循笔记记录的宗旨（**记录理解和方便复习**），可以任意选择

## 示例

这里阅读《How to read a paper》这篇论文为例

### The first pass

首先我们读文章标题、摘要、引言，小标题和结论，可以意识到这篇论文主要是介绍了一种读论文的方法。然后我们接着读references，很不幸，这里没有一篇参考文献我们读过，所以不做任何标记

读完第一遍，我们来回答五个问题：这篇文章是介绍性的，相关的工作有3篇，不需要任何理论基础即可阅读。文章没有任何假设，并且有条理地表达了他们的工作。

思考之后，这篇文章有利于帮助我们学习如何阅读论文，是对我们有价值的文章，值得阅读

### The second pass

我们接着看文中的图表，这里没有，继续下一步。参考文献里边都是没有阅读过的工作，统统标记。

读完第二遍的话，我们来尝试把握这篇文章的脉络：文章首先提出了读论文的重要性和有效读论文的必要性。第二章节顺序介绍了THREE-PASS方法的三个步骤，包括每一个步骤需要达成的目的。第三章节主要介绍了如何寻找值得阅读的论文。第四实验部分主要讲述了作者自身的经历（从这篇文章的小标题能比较好地推断出段落大意）

这里我们发现自己好像能概述文章的内容，所以直接进行第三遍阅读

### The third pass

最后一遍，我们深入细致地阅读论文。本文示例章节之前的内容均可视作third
pass的分析总结。并且我们在阅读完整篇论文之后能发现这篇论文介绍的方法更多适合读一些具体的工作，而不是综述性质的文章。文章条理清楚地介绍了每一遍阅读论文的做法，并且有实验部分，总体结构完备。

总结：这篇论文能指导我们掌握阅读论文的方式，并且引用了介绍如何对文章做review和写一篇科技文章的工作。

## 工具推荐

- [alphaxiv](https://www.alphaxiv.org/)：利用LLM快速阅读arXiv上的论文
- [zotero](https://www.zotero.org/)：具备丰富插件生态的文献管理与阅读工具

笔者使用zotero进行文献管理，zotero的分类有两种方式：collection和tag。笔者用collection表示文献涉及到的领域，tag表示文献具体应用到的技术。比如《SSFuzz: State-Guided Fuzzing With Shared Feedback for Black-Box IoT Devices》这篇论文，笔者会将其放入IoT群组，设置标签为Fuzz

这里再推荐一些笔者常用的zotero插件：

1.  Better BibTex for Zotero：方便导出参考文献

2.  Ethereal Style：能够设置样式（常用的应该是标签功能和笔记功能）

3.  Zotero Focused Mode：能够隐藏导航栏，沉浸式阅读文章

4.  Zotero Reading List：能将文献设置为5个不同的状态（New, To Read, In
    progress, Read, Not Reading），不用再为记不住读过哪篇文献而烦恼

强烈推荐使用zotero的浏览器插件，一键导入真的香
